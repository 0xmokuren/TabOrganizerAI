# スクリーンショット撮影ガイド

Chrome Web Store の必須項目はスクリーンショット **最低 1 枚**、推奨 **5 枚**。サイズは **1280x800** または **640x400**（PNG / JPEG）。1280x800 がトリミング自由度高くておすすめ。

## 事前準備

1. プロファイルをクリーンにする（個人情報を写さないために新規プロファイル推奨）
   - Chrome 右上アバター → 「追加」→ 新規ユーザー作成
2. その新規プロファイルに `tab-cluster-ai-1.5.2.zip` を展開して読み込む
   - `chrome://extensions` → デベロッパーモード ON → 「パッケージ化されていない拡張機能を読み込む」
3. デモ用にタブを 10〜15 個開く（下の「おすすめのデモタブ構成」参照）
4. macOS なら ⌘+Shift+4 → Space でウィンドウキャプチャ。Chrome ウィンドウを 1280x800 にリサイズしておくと楽
   - Chrome DevTools のデバイスモードで Custom 1280x800 にすると正確

## 撮影すべき 5 枚

| # | 画面 | 撮り方 |
| --- | --- | --- |
| 1 | **Before/After** — 散らかったタブバー → 整理済みのタブグループ | Chrome ウィンドウ全体（タブバーが見える状態）。横並び合成して 1280x800 に。最強の 1 枚 |
| 2 | **AI 提案プレビュー** | ポップアップで「AI で分析」をクリックし、グループ名と色付きプレビューが並んだ瞬間。ポップアップ + 後ろのタブバーが見える構図 |
| 3 | **ポップアップ初期画面** | プロバイダ切替（On-device / Gemini API）と「Analyze with AI」「Organize by domain」ボタンが見える状態 |
| 4 | **Gemini API 設定 + モデル選択** | API キー欄（key は伏せる）、モデルドロップダウンを開いた状態 |
| 5 | **適用後のタブグループ** | 色付きグループが Chrome タブバーに並んだ完成形。グループ名が読める拡大気味の構図 |

> 5 枚難しければ 1・2・5 の 3 枚でも十分。1 枚しか撮らないなら #1（Before/After）が一番訴求力高い。

## 文字を入れたい場合（任意）

各ショットに簡単なキャプションを乗せると CVR が上がりやすい:

- #1: 「散らかったタブを 1 クリックで整理」
- #2: 「AI の提案を確認してから適用」
- #3: 「オンデバイス AI または Gemini API」
- #4: 「複数の Gemini モデルから選択」
- #5: 「Chrome ネイティブのタブグループに自動分類」

Figma / Canva / Keynote のテキストオーバーレイで十分。ブランドカラーは `icons/icon128.png` の青系に合わせると統一感が出る。
